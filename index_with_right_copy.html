DevTools — Local-First API Flow Testing

A free, open-source API client and flow runner. Record browser traffic, turn it into a flow tree with auto-mapped variables, export to YAML, and run at multithreaded Go speed locally or in CI.

curl -fsSL https://sh.dev.tools/install.sh | bash

What you get

  • Record real HAR traffic and instantly see a flow tree with variables auto-mapped.
  • Export flows as YAML so requests become reviewable artifacts in Git.
  • Run locally or in CI with a Go-based runner that executes steps in parallel.
  • No accounts, no telemetry. Everything stays on your machine.

Why DevTools

Unlike modern API tools, DevTools helps you ship reliable APIs with exportable YAML and a local-first workflow that respects your team’s velocity.

The flow from traffic to tests in minutes

  • HAR → Flow Tree: import a HAR from your browser and instantly get a visual flow graph.
  • Auto Variable Mapping: tokens/IDs are extracted and addressable by node, e.g. {{ login.response.body.token }}.
  • Flows as YAML: exportable text files so you can reuse a saved request and override headers/body/params/assertions per node—no duplication.
  • Go-Fast Runner: execute flows in parallel with a lean, multithreaded Go engine.
  • Local-First: no accounts, no cloud sync, telemetry off by default.

Result: record real traffic, model behavior, and run thousands of API checks in seconds—locally and in CI.

Hello API (2 minutes)

  1. Install the CLI or download the desktop app.
  2. Create a request to https://jsonplaceholder.typicode.com/todos/1.
  3. Add assertion: response.status == 200.
  4. Send. Inspect JSON, status, time, size.
  5. Add a login request, capture the token, and use Authorization: Bearer {{ login.response.body.token }}.

Example: YAML Flow

Exported or hand-authored flows are plain text.

flows:
  - name: ecommerce_smoke
    steps:
      - request:
          name: login
          use_request: login
          overrides:
            body:
              email: admin@example.com
              password: secret
      - request:
          name: list_categories
          use_request: get_categories
          overrides:
            headers:
              authorization: "Bearer {{ login.response.body.token }}"
            assertions:
              - "response.status == 200"
      - request:
          name: create_product
          use_request: post_products
          overrides:
            headers:
              authorization: "Bearer {{ login.response.body.token }}"
            body:
              name: "macbook pro"
              price: 123
              category_id: "{{ list_categories.response.body[0].id }}"
            assertions:
              - "json.body.id != null"

Run in CI: devtools run ./flows/ecommerce_smoke.yaml --report console --env Dev

Testing Philosophy — The DevTools Manifesto

  1. Make API flows the backbone. UI test sprawl is brittle and slow. Keep end-to-end UI to 3–5 happy paths; move the rest into API flow tests and component tests.
  2. Visual when you want it, code when you need it. Author in a visual builder, then export to YAML so the exact behavior is diff-able and gateable in CI.
  3. Real traffic is truth. Start from recorded HARs to reflect what users actually do; turn that into executable flows with auto-mapped variables.
  4. Speed unlocks discipline. A parallel Go runner makes it practical to run thousands of checks on every PR. Fast feedback creates better habits.
  5. Local-first earns trust. Your requests, data, and credentials stay on your machine. No third-party servers needed to test your APIs.

This is the pragmatic, scalable path we’ve learned building and operating large test systems.
